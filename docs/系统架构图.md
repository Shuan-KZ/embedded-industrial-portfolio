# 系统架构

## 五层结构

```
┌─────────────────────────────────────────────────┐
│  PLC层     S7-1200 + SCL程序(DB1/FC1/OB1)       │
│            Modbus TCP :502                      │
├─────────────────────────────────────────────────┤
│  上位机层   C# WinForm + S7.NET                  │
│            仿真/真实双模式，POST转发到后端         │
├─────────────────────────────────────────────────┤
│  后端层    SpringBoot :8081                      │
│            REST API + MyBatis + H2内嵌数据库      │
│            统一Result返回 + 参数校验 + 全局异常    │
├─────────────────────────────────────────────────┤
│  算法层    Flask :5000                           │
│            IMOGJO多目标优化 + 设备状态融合         │
│            权重调节 + 有/无约束对比               │
├─────────────────────────────────────────────────┤
│  大屏层    ECharts 5.5 暗色工业风                 │
│            4个Tab：监控/OEE/调度/系统控制          │
└─────────────────────────────────────────────────┘
```

## 数据怎么走的

```
PLC DB1 → Modbus TCP → C# S7.NET读取 → HTTP POST到SpringBoot
→ MyBatis写入H2 → 大屏3秒轮询GET /api/devices → ECharts刷新图表
```

```
点"重新调度" → POST到Flask(带权重参数)
→ Flask调SpringBoot拿设备状态 → 故障剔除/预警降速
→ IMOGJO跑一遍 → 返回甘特图JSON → 前端custom series渲染
```

## 各层技术选型

| 层 | 用了什么 | 为什么选它 |
|----|---------|-----------|
| PLC | S7-1200 + SCL | 学校实验室有，工业现场主流 |
| 上位机 | C# WinForm + S7.NET | .NET对西门子支持好，WinForm开发快 |
| 后端 | SpringBoot + MyBatis | SSM课堂学的，直接上手 |
| 数据库 | H2 内嵌 | 零安装，文件存储 |
| 算法 | Python + Flask | NumPy矩阵运算方便，Flask轻量 |
| 大屏 | ECharts | 图表类型全，文档好，CDN引入零配置 |
| 启动器 | Python Rich | 终端美化，pip install即可 |
