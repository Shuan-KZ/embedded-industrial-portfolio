## 子项目3：SpringBoot后端 ✅

### 启动
```bash
cd monitor_server/demo && mvnw spring-boot:run
```

### API清单
| 接口 | 说明 |
|------|------|
| GET /api/devices | 6设备+最新数据+统计 |
| POST /api/device-data | 接收上位机批量数据 |
| GET /api/device-data/range | 历史曲线数据 |
| GET /api/alarms | 报警列表 |
| PUT /api/alarms/{id}/resolve | 故障恢复 |
| GET /api/oee | OEE设备综合效率 |
| GET /api/export/daily-report | Excel日报导出 |

### 数据库
- H2 内嵌数据库，文件存储于 data/ 目录
- 表：device_info(6台设备), device_data(时序), alarm_record(报警)

## 子项目4：Web大屏 ✅

### 访问
`http://localhost:8081/dashboard.html`

## 子项目5：算法微服务 ✅

### 启动
```bash
cd 05_算法服务 && python scheduler.py
```

### 接口
| 接口 | 说明 |
|------|------|
| POST /api/optimize | 调度优化（设备状态融合+无约束对比） |
| GET /api/health | 健康检查 |

### 融合逻辑
- GET后端 /api/devices 拿实时设备状态
- status=1(预警) → 加工时间×1.2
- status=2(故障) → 该机器从可选集剔除

## 一键启动

```bash
python launcher.py          # 终端交互菜单
python launcher.py start    # 直接启动全部
```
或双击 `start.bat`

## 全链路数据流

```
PLC DB1 → C# S7.NET读取 → HTTP POST到SpringBoot
→ MyBatis写入H2 → 大屏3秒轮询GET /api/devices → ECharts刷新图表

点"重新调度" → POST到Flask
→ Flask调SpringBoot拿设备状态 → 故障剔除/预警降速
→ IMOGJO优化 → 返回甘特图 → 前端渲染
```
