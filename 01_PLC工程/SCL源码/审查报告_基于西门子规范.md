## ❌ 严重问题

### 1. 能耗系数计算错误 — OB1:22-24

**代码**：
```
// 采样间隔(2秒)换算为小时: 2/3600 = 1/1800
#功率系数 := 2.777e-4;  // = 1/3600 秒→小时换算
```
**问题**：注释说采样间隔2秒(1/1800≈5.556e-4)，但代码用了1/3600(≈2.777e-4)。能耗累计会少一半。

**修复**：统一为 `1/1800 ≈ 5.556e-4`，或直接用公式表达。

### 2. 无限循环累加 — OB1 无定时门控

**问题**：OB1 每个扫描周期都执行，FOR循环每次扫描都给6台设备累加能耗。S7-1200扫描周期毫秒级，几秒钟能耗就飞了。

**修复**：加一个2秒定时器门控，只在定时器触发时才累加。

## ⚠️ 主要问题

### 3. 缺少块头部信息
规范要求每个块都应有：`TITLE`、`AUTHOR`、`FAMILY`、`VERSION`。三个块都缺。

### 4. 魔数硬编码
`60.0/80.0/4.0/7.0/3.5` 这些阈值和参数应定义为 CONSTANT：
```
VAR CONSTANT
    TEMP_WARN_THRESHOLD : Real := 60.0;
    TEMP_FAULT_THRESHOLD : Real := 80.0;
    VIB_WARN_THRESHOLD : Real := 4.0;
    VIB_FAULT_THRESHOLD : Real := 7.0;
    RATED_POWER_KW : Real := 3.5;
END_VAR
```

### 5. 循环计数器数据类型
规范要求循环计数器用 DINT：`#循环计数 : Int` → 应改为 `DINT`。

### 6. OB1 变量命名
规范要求临时变量加 `_t_` 前缀（如 `_t_设备编号`）。虽然中文变量名在TIA Portal完全可用，但临时变量的前缀能一眼区分生命周期。

## ℹ️ 建议优化

### 7. 缺少 REGION 分段
建议用 REGION 把代码按逻辑分段：
- REGION 初始化
- REGION 循环处理6设备
- REGION 阈值判定

### 8. `S7_Optimized_Access := 'FALSE'` 可加注释
这里关掉优化访问是为了保证DB1绝对地址固定（C# S7.NET 通过偏移量读取），**是有意为之**，但应写在注释里，否则别人会以为不规范。

### 9. CASE 缺少 ELSE 分支
`设备编号` 如果不是1-6，当前代码什么都不做。加 ELSE 分支可防止边界错误。

### 10. 温度+振动双重判定逻辑隐患
FC1 被调用两次：
- 第1次判温度（故障自锁=TRUE），第2次判振动（故障自锁=FALSE）
- 如果温度判为故障(#新状态=2)，振动判正常 → #新状态被覆写为0
- 因为两次 FC1 调用共用同一个 `#临时状态` 变量，第2次调用会覆盖第1次结果

**这是一个潜在的逻辑bug**：温度故障会被振动判定覆盖掉。需要确认这是否为预期行为。
